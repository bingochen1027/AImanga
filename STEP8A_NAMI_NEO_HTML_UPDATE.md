# Step 8A｜NamiStory + Neo 功能融合 HTML 更新说明

## 本轮目标

在 Step 6 的中文化与付费闭环版本基础上，把 NamiStory 的“分阶段明盒产线”和 Neo / Neowow 的“Skill 化创作机制”落到现有 HTML 原型中。

本轮不新增图片、不改价格主体系、不扩展外部调研，只优化核心用户路径：

> 首页选择入口 → 剧本向导 → 工作台 → 编辑器 Guide → 个人中心

---

## 核心借鉴

### 1. NamiStory：明盒式创作产线

产品不直接让用户生成最终视频，而是拆成：

1. 概念
2. 剧本
3. 资产
4. 分镜
5. 视频
6. 后期
7. 投放

每个阶段都要有确认点、风险提示、产出物和积分预估。

### 2. Neo / Neowow：Skill 化创作体验

把“剧本框架、内容填充、角色资产、分镜导演、投放素材、合规审核”等能力包装成可理解、可启用、可升级的 Skill，让用户知道 AI 在每一步到底帮他做什么。

---

## 本轮已更新页面

### 1. `index.html` 首页

新增：

- “我有完整剧本”入口
- “我只有一个想法”入口
- “我想做同款模板”入口
- 内置创作 Skill 模块

目标：把首页从展示型官网升级为创作路径入口。

---

### 2. `features.html` 产品功能页

新增：

- “阶段 + Skill + 产出物”七阶段产线
- 概念 / 剧本 / 资产 / 分镜 / 视频 / 后期 / 投放

目标：让功能页不只是罗列能力，而是解释每一步的操作逻辑。

---

### 3. `wizard.html` 剧本向导

新增：

- 三条路径：已有剧本 / 只有想法 / 从模板开始
- 7 层问答式“剧本框架 Skill”
- 结构化输出示例
- 成本闸门说明

目标：让没有完整剧本的用户也能进入创作流程。

---

### 4. `studio.html` 创作工作台

新增：

- 阶段导航：概念、剧本、资产、分镜、视频、后期、投放
- 下一步建议
- 缺失资产提示
- 预计积分说明
- 跳转 `editor-guide.html` 的入口

目标：让工作台变成项目生产总控，而不是普通项目列表。

---

### 5. `account.html` 个人中心

新增：

- 我的技能
- 推荐技能
- 项目生产进度
- 资产健康度
- 下一步任务

目标：把个人中心从账号/钱包页升级成创作者经营中心。

---

### 6. 新增 `editor-guide.html`

新增页面，用于模拟真正编辑器体验。

页面结构：

- 左侧：阶段导航
- 中间：当前阶段编辑区
- 右侧：AI Guide 面板
- 底部/卡片：成本闸门、风险提示、下一步按钮

目标：把 NamiStory 的“阶段可见、可控、可回退”和 Neo 的“Skill Guide”结合成一个核心编辑页面。
