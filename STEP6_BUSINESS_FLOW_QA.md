# Step 6 业务闭环 QA 说明

## 本轮目标

本轮不新增页面、不新增图片，重点完成：

1. 全站用户可见英文文案中文化。
2. 价格页与实际付费操作对齐。
3. 订单确认页更像真实付款页。
4. 支付成功页能自然回到个人中心。
5. 个人中心能承接会员、积分、项目包、订单、发票和商用授权。
6. 抖音投放包页面能说明“购买投放包后可批量生成”的权益边界。

## 已完成修正

### 1. 中文化检查
- 10 个 HTML 页面用户可见文本已完成中文化。
- 保留文件名、代码属性、路径参数中的英文，这是静态原型运行所需，不对用户展示。
- 图片内部的英文或品牌字样属于图片内容，未在本轮重绘；页面文案已用中文覆盖关键业务信息。

### 2. 价格体系对齐
价格页统一为五类付费方式：

| 类型 | 页面入口 | 后续动作 |
|---|---|---|
| 免费试用 | 登录 / 注册 | 领取试用额度，不进入支付 |
| 订阅会员 | 价格页 / 个人中心 | 进入订单确认页 |
| 积分充值包 | 价格页 / 个人中心 | 进入订单确认页 |
| 项目包 / 投放包 | 价格页 / 投放包页 / 个人中心 | 进入订单确认页 |
| 企业版 | 价格页 | 进入订单确认页并提交企业咨询记录 |

## 验证结果

- 图片引用：无缺失。
- 内部 HTML 链接：无缺失。
- 用户可见英文文本：已清理为中文。
- 付费路径：价格页 / 个人中心 / 投放包页 → 订单确认 → 支付成功 → 个人中心。

## 建议 Review 顺序

1. `pricing.html`：确认五类付费方式是否清楚。
2. `checkout.html?plan=creator`：确认订阅会员付款页。
3. `checkout.html?plan=credits`：确认积分充值付款页。
4. `checkout.html?plan=project`：确认项目包 / 投放包付款页。
5. `payment-success.html?plan=project`：确认支付后状态。
6. `account.html`：确认权益回写与经营中心闭环。
7. `launch-kit.html`：确认投放包购买和实际生成能力一致。
