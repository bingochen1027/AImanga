产品规划底稿｜三端实体 + 最小闭环

官网、Portal、NeoTV&Neowow 的功能清单与数据模型

目标是先拆出最小可上线单元：官网负责获客与创作入口，Portal 负责数据查看与用户权限，NeoTV&Neowow 负责案例内容、技能能力与生成服务供给。

P0：必须上线，形成闭环 P1：可上线后增强转化与效率 P2：锦上添花或后续商业化 版本：MVP 规划稿

一、三个实体的边界

先把责任边界切清楚，避免官网、后台和能力平台互相长出重复功能。

实体 1｜官网

面向创作者的前台产品

承担访问、理解、注册、模板/案例浏览、创作、购买、个人中心等用户旅程。

  • 展示产品价值、案例和价格
  • 引导免费生成第一集
  • 承载模板中心、案例中心、工作台、投放包
  • 展示项目、积分、订单、授权与资产
P0 转化闭环P1 内容增强
实体 2｜Portal

面向内部运营的后台

第一阶段不做复杂工单，只做数据查看、今日对比、用户启禁用、权限与用户导出。

  • 查看注册、项目、生成、支付数据
  • 今日 vs 昨日的核心指标对比
  • 用户角色、启用/禁用、全部字段导出
  • 生成失败、待复核状态只记录不处理
P0 只读看板P0 权限管理
实体 3｜NeoTV&Neowow

内容案例与能力供给端

NeoTV 提供可展示的案例内容和数据样本；Neowow 提供技能、工作流、生成任务和结果回传。

  • 案例库、封面、标签、题材和展示素材
  • 剧本、分镜、投放、合规等创作技能
  • 生成任务、状态、成本、失败原因
  • 输出资产、项目档案、投放素材包
P0 技能调用P1 案例同步

最小可上线闭环

最小单元不是“所有页面都完整”，而是用户能完成一次真实创作意图，后台能看到这次行为，能力端能返回结果。

1 官网访问首页 / 案例 / 模板 / 价格
2 注册登录领取试用积分
3 创建项目一句话、模板或案例同款
4 调用技能剧本策划 P0，其他阶段可占位
5 返回结果项目档案、消耗、任务状态
6 后台可见数据看板、用户状态、导出

三、核心数据表结构

表结构按“用户/权限、项目/创作、交易/积分、内容/能力、运营日志”五组组织。

users 用户表 P0

官网与 Portal 共用的用户主表。

  • id用户唯一编号
  • phone / email手机号、邮箱
  • name / avatar昵称与头像
  • status启用、禁用、注销
  • source_channel首页、模板、案例、投放包
  • created_at / last_active_at注册与最近活跃时间

roles / user_roles 权限表 P0

Portal 用于角色控制，官网用于拦截禁用用户。

  • role_id超级管理员、运营管理员、只读运营、创作者
  • permission_keys看板查看、用户管理、导出用户
  • user_id绑定用户
  • assigned_by操作管理员
  • assigned_at授权时间

projects 项目表 P0

创作工作台的核心业务对象。

  • id项目编号
  • owner_user_id项目归属用户
  • title / synopsis项目名、故事简介
  • source_type一句话、剧本、模板、案例同款
  • current_stage剧本、视觉、资产、分镜、视频、后期、投放
  • progress / status进度与状态

project_steps 阶段步骤表 P0

用于工作台七阶段与每阶段内部步骤。

  • project_id所属项目
  • stage_keyscript、visual、asset、storyboard、video、preview、launch
  • step_key当前步骤标识
  • input_snapshot用户选择与输入快照
  • output_artifact_id产物编号
  • status未开始、进行中、完成、失败

creative_jobs 生成任务表 P0

官网发起，Neowow 执行，Portal 查看。

  • id任务编号
  • project_id / user_id项目与用户
  • skill_id / workflow_id调用的技能与流程
  • status排队、运行、成功、失败、待复核
  • credit_cost预计/实际积分
  • error_code / retry_count失败原因与重试次数

artifacts / assets 产物资产表 P0/P1

保存剧本、项目档案、图片、封面、视频、投放素材。

  • artifact_id产物编号
  • project_id所属项目
  • type剧本、角色、分镜、视频、封面、投放包
  • content_url / json_payload文件地址或结构化内容
  • version版本号
  • source_job_id由哪个任务产生

credit_wallets / credit_ledger 积分表 P0

支持试用积分、会员赠送、充值、消耗、失败返还。

  • wallet_id / user_id钱包与用户
  • balance当前余额
  • ledger_type发放、购买、消耗、返还
  • amount积分变化数量
  • related_order_id / job_id关联订单或任务

orders / products 交易表 P0

价格页、订单确认、支付成功和个人中心使用。

  • product_id会员、积分包、项目包、投放包
  • order_id / user_id订单与用户
  • amount / status金额与支付状态
  • entitlement_snapshot购买后权益快照
  • invoice_status发票状态

cases / templates 案例模板表 P0

官网案例中心与模板中心读取，也可来自 NeoTV。

  • case_id / template_id案例或模板编号
  • title / category标题、题材分类
  • cover_url封面图
  • method_summary制作方法、适合人群
  • sourceNeoTV、内部上传、运营配置
  • status草稿、上线、下线

skills / workflows 技能流程表 P0/P1

来自 Neowow 的技能定义和工作流版本。

  • skill_id剧本创作、分镜、投放、合规
  • version技能版本
  • input_schema输入结构
  • output_schema输出结构
  • cost_rule积分预估规则
  • status可用、停用、灰度

events / audit_logs 事件日志表 P0

Portal 的数据看板和权限审计依赖。

  • event_id事件编号
  • user_id / session_id用户或访客会话
  • event_name访问、注册、创建项目、购买、生成
  • payload事件详情
  • created_at发生时间

exports 导出记录表 P0

Portal 导出用户字段需要留痕。

  • export_id导出编号
  • operator_user_id操作人
  • export_type全部用户、订单、项目等
  • field_scope导出字段范围
  • file_url文件地址
  • created_at导出时间

五、三个实体的数据传输模型

以下是三端之间最小需要交换的数据，不是接口命名，而是后端边界。

官网 → Portal

访问与转化事件页面访问、按钮点击、注册、模板选择、案例点击、价格点击
用户与权益数据用户信息、来源渠道、会员状态、积分余额、禁用状态同步
项目与任务状态项目创建、当前阶段、生成任务、失败/待复核状态
订单与授权订单、支付状态、发票、商用授权记录

官网 ↔ NeoTV&Neowow

案例与模板读取案例图片、题材、创作方法、模板预设
技能任务请求项目上下文、用户选择、目标平台、预算与输出要求
任务状态回传排队、运行、成功、失败、预计耗时、积分消耗
创作产物回传剧本方向、大纲、项目档案、分镜、封面、视频和投放素材

Portal ↔ NeoTV&Neowow

能力健康状态技能是否可用、失败率、平均耗时、队列积压
内容供给状态案例上线/下线、模板版本、素材来源
异常记录查看失败任务、待复核任务、失败原因和重试次数
后续运营配置P1 后可配置案例排序、模板推荐、技能灰度

七、开发拆分建议

按最小闭环，后端先做少量稳定表与接口，前端先跑通真实状态流。

P0 第一阶段

  • 用户注册登录、状态启禁用
  • 官网页面与创建项目入口
  • 剧本策划技能任务调用
  • 项目档案与任务结果保存
  • 积分发放、扣减、失败返还
  • Portal 只读看板、用户管理、导出

P1 第二阶段

  • 案例与模板后台配置
  • 投放包生成与项目资产关联
  • 更多创作阶段接入：视觉、资产、分镜
  • 内容合规记录与授权材料下载
  • 能力健康看板与失败率统计

P2 后续增强

  • 团队空间与多人协作审核
  • 工单、退款、补偿后台
  • 技能市场与工作流编排器
  • 投放数据回流与智能复盘
  • 企业级私有资产库和接口对接
