# 智能漫剧网站与运营后台功能及开发需求清单

## 1. 文档目的

本清单用于统一产品、设计、前端和后端的开发范围，明确：

- 网站每个页面承担什么任务；
- 页面之间如何串联；
- 前端需要实现哪些交互和状态；
- 后端需要提供哪些数据、接口和业务规则；
- 哪些内容属于当前原型，哪些内容需要进入正式开发。

后续可基于本文档的“后端需求”章节，形成提供给智灵新境的正式技术对接文档。

---

## 2. 产品主流程

用户主流程：

> 首页／案例中心／模板中心 → 登录注册 → 项目创建向导 → 引导式创作台 → 创作工作台 → 抖音投放包 → 价格与订单 → 个人中心 → 继续下一集

运营主流程：

> 用户来源 → 注册试用 → 项目创建 → 创作阶段 → 智能任务 → 积分消耗 → 素材导出 → 订单支付 → 授权交付 → 复购与续作

核心原则：

1. 每个页面只承担一个主要决策。
2. 项目共用一条七阶段状态主线。
3. 轻松创作、进阶控制、专业编排只改变控制深度，不创建不同项目副本。
4. 所有高成本生成动作必须先显示预计产物、耗时和积分。
5. 所有确认结果必须有版本记录，回退不清空旧结果。

## 3. 网站页面功能与开发需求

### 3.0 网站页面总表

| 页面 | 主要任务 | 核心功能 | 上一步 | 下一步 |
|---|---|---|---|---|
| 首页 | 建立信任并开始转化 | 价值横幅、案例、流程、免费生成 | 外部来源 | 案例、模板、登录、项目向导 |
| 产品功能 | 解释七阶段能力 | 阶段导航、能力卡、适用人群 | 首页 | 引导式创作台 |
| 模板中心 | 提供快速创作起点 | 题材筛选、模板详情、做同款 | 首页 | 项目向导 |
| 案例中心 | 用作品证明能力 | 案例筛选、预览、风格参考 | 首页 | 项目向导 |
| 登录注册 | 建立用户身份 | 手机号、验证码、试用权益 | 首页或购买页 | 原目标页 |
| 项目创建向导 | 创建项目 | 三种起点、剧本上传、成本预估 | 登录、模板、案例 | 引导式创作台 |
| 引导式创作台 | 完成关键创作决策 | 三种深度、方向、大纲、审稿、项目档案 | 项目向导 | 创作工作台 |
| 创作工作台 | 继续生产与处理任务 | 当前任务、项目阶段、队列、预算 | 引导式创作台 | 创作说明、投放包 |
| 创作说明 | 解释当前阶段 | 资产、分镜、风险、待确认问题 | 工作台 | 对应创作步骤 |
| 抖音投放包 | 生成发布素材 | 标题、封面、钩子、组合、导出 | 工作台 | 价格或个人中心 |
| 价格 | 选择商业方案 | 会员、积分、项目包、企业咨询 | 全站购买入口 | 确认订单 |
| 确认订单 | 创建支付 | 商品、权益、支付方式 | 价格 | 支付成功 |
| 支付成功 | 确认权益到账 | 订单号、到账结果、下一步 | 确认订单 | 个人中心 |
| 个人中心 | 留存与资产管理 | 项目、资产、钱包、订单、授权、团队 | 支付成功、工作台 | 继续下一集 |

### 3.1 首页

**页面目标**

快速说明产品价值，以案例建立信任，并引导用户查看作品或免费生成第一集。

**功能清单**

- 产品核心价值横幅；
- 免费生成第一集按钮；
- 查看案例按钮；
- 案例瀑布流；
- 题材筛选展示；
- 三步创作流程；
- 产品功能、模板、案例、价格、个人中心、工作台、抖音投放包导航；
- 日间与暗夜模式；
- 登录入口。

**前端开发要求**

- 案例卡响应式布局；
- 顶部导航在桌面端完整展示，移动端折叠菜单；
- 主题状态本地保存并在页面间同步；
- 免费生成按钮根据登录状态跳转到登录或项目向导；
- 记录案例曝光、案例点击和主要按钮点击事件；
- 图片使用统一比例、懒加载和加载失败占位图。

**后端需求**

- 首页配置接口：横幅文案、按钮、推荐案例和创作流程；
- 案例列表接口：标题、封面、题材、作者、形式、排序、发布状态；
- 用户登录状态与试用权益查询；
- 页面曝光、来源和按钮事件接收；
- 内容管理后台支持推荐位和案例排序。

**验收标准**

- 新用户可在两次点击内进入创建项目；
- 所有图片有替代文本和错误占位；
- 桌面和移动端无页面横向溢出。

### 3.2 产品功能页

**页面目标**

按七个创作阶段解释完整生产能力，而不是罗列孤立工具。

**功能清单**

- 剧本策划；
- 视觉设定；
- 角色、场景与道具资产；
- 分镜脚本；
- 视频生成；
- 后期预览；
- 发布投放；
- 数据复盘；
- 合规与授权。

**前端开发要求**

- 阶段导航与当前标签状态；
- 能力卡片按阶段切换；
- 每项能力显示输入、输出、适用人群和下一步；
- 页面按钮直接进入引导式创作台。

**后端需求**

- 产品能力配置接口；
- 不同会员等级可用能力和限制查询；
- 能力运行状态、预计积分和预计耗时查询。

### 3.3 模板中心

**页面目标**

让用户从题材模板或创作方法模板快速开始项目。

**功能清单**

- 题材筛选；
- 模板卡；
- 适用人群与推荐集数；
- 预估价格和创作深度；
- 做同款；
- 查看案例；
- 创作工作流模板。

**前端开发要求**

- 筛选和排序；
- 模板详情抽屉或详情页；
- 选择模板后预填项目向导；
- 明确展示模板会预填和锁定哪些字段；
- 模板收藏与最近使用状态。

**后端需求**

- 模板列表、详情、分类、标签和排序接口；
- 模板预填参数和依赖能力查询；
- 模板版本管理；
- 模板收藏、使用次数和转化数据；
- 运营后台模板发布、下架和推荐管理。

### 3.4 案例中心

**页面目标**

展示不同题材和交付形式，并支持用户从案例创建相似项目。

**功能清单**

- 题材筛选；
- 案例作品墙；
- 播放、完播、互动等示例指标；
- 案例制作过程；
- 设为风格参考；
- 从案例开始。

**前端开发要求**

- 案例筛选与详情；
- 视频或图片预览；
- “从案例开始”将题材、平台和视觉方向写入项目向导；
- 风格参考必须让用户逐条确认，不直接复制不可解释参数。

**后端需求**

- 案例内容与媒体资源接口；
- 案例关联模板和可提取风格规律；
- 案例授权范围和可否用于参考的标记；
- 案例浏览、收藏和转化数据。

### 3.5 登录注册页

**页面目标**

以最低阻力完成登录，并发放试用权益。

**功能清单**

- 手机号登录；
- 验证码发送与校验；
- 微信登录预留；
- 用户协议和隐私政策；
- 试用积分说明；
- 登录后返回原目标页面。

**前端开发要求**

- 手机号和验证码表单校验；
- 发送验证码倒计时；
- 登录错误和频率限制提示；
- 登录态安全保存；
- 支持来源页面回跳。

**后端需求**

- 验证码发送和校验；
- 用户注册、登录、退出和登录态刷新；
- 微信身份绑定；
- 用户协议版本确认记录；
- 首次试用积分发放及防重复规则；
- 登录安全、频率限制和异常设备记录。

### 3.6 项目创建向导

**页面目标**

让用户从一句话、完整剧本或模板三种起点创建项目。

**功能清单**

- 只有一个想法；
- 已有完整剧本；
- 从模板开始；
- 故事输入；
- 文档上传；
- 平台、受众、题材、时长和集数；
- 内容要素选择；
- 镜头、积分和时间预估；
- 创建项目。

**前端开发要求**

- 分步表单与草稿保存；
- 文件上传进度、格式和大小提示；
- 项目成本动态预估；
- 已登录用户直接创建，未登录用户保留表单后登录；
- 创建成功后进入引导式创作台。

**后端需求**

- 项目创建、读取和更新；
- 剧本文本与文档上传、解析和安全扫描；
- 模板参数应用；
- 项目成本与耗时预估；
- 项目编号、默认版本和七阶段状态初始化；
- 项目创建幂等控制，避免重复项目。

### 3.7 引导式创作台

**页面目标**

通过轻松创作、进阶控制和专业编排三种深度，引导用户完成关键创作决策。

**功能清单**

- 七阶段项目产线；
- 三种创作深度切换；
- 单题任务；
- 创作方向卡；
- 双版本大纲对比；
- 多视角审稿；
- 项目档案；
- 风格规律；
- 上下文、模型路由、预算与失败策略的专业控制；
- 智能助手说明当前步骤；
- 保存、回退、锁定和进入下一步。

**前端开发要求**

- 同一项目三种深度共用状态；
- 每一步显示目标、输入、预计输出、风险、耗时和积分；
- 自动保存并显示保存状态；
- 返回旧步骤时创建新版本，不覆盖旧稿；
- 生成任务使用轮询或实时推送更新状态；
- 失败、取消、积分不足和人工复核状态完整；
- 页面刷新后恢复当前阶段和当前问题；
- 专业控制折叠后不得丢失参数。

**后端需求**

- 项目阶段状态机；
- 创作问题、选项和推荐理由配置；
- 智能任务创建、排队、执行、取消、重试和查询；
- 方向、大纲、审稿、项目档案的结构化输出；
- 项目版本、分支、锁定和回退；
- 上下文与资产依赖；
- 模型选择和路由策略；
- 任务预算、积分预扣、成功扣减、失败返还；
- 运行日志、失败原因和人工复核；
- 不同会员和团队角色的能力权限。

### 3.8 创作工作台

**页面目标**

集中展示项目状态、当前任务、生成队列、预算和待确认事项。

**功能清单**

- 当前项目；
- 继续当前创作；
- 新建项目；
- 项目阶段；
- 待确认事项；
- 生成队列；
- 项目预算；
- 项目看板；
- 深色编辑器入口。

**前端开发要求**

- 默认把最需要处理的任务放在首位；
- 项目卡支持按阶段、更新时间和异常筛选；
- 队列状态实时更新；
- 进入项目时恢复具体步骤，而不是只进入项目首页。

**后端需求**

- 用户项目列表与项目摘要；
- 待办和异常事项；
- 任务队列汇总；
- 项目预算、已用积分和预计剩余；
- 最近访问和继续位置。

### 3.9 创作说明页

**页面目标**

解释当前项目阶段、已完成内容、待确认内容和下一步。

**功能清单**

- 七阶段说明；
- 当前阶段；
- 已生成资产；
- 待确认资产；
- 分镜预览；
- 风险提示；
- 进入当前阶段。

**前端开发要求**

- 根据项目状态动态展示；
- 卡片可跳转到具体资产、镜头或问题；
- 不展示与当前阶段无关的技术参数。

**后端需求**

- 项目阶段摘要；
- 角色、场景、道具和分镜资产查询；
- 一致性、完整度和风险评分；
- 待确认问题列表。

### 3.10 抖音投放包

**页面目标**

围绕已有正片生成可测试的标题、封面、开场钩子和评论引导。

**功能清单**

- 投放封面；
- 标题建议；
- 开场钩子；
- 组合建议；
- 免费预览；
- 完整投放包解锁；
- 多版本素材；
- 评论引导；
- 复盘建议；
- 素材导出和授权记录。

**前端开发要求**

- 从一键推荐逐步开放自由组合和批量矩阵；
- 显示免费与付费范围；
- 生成前显示积分与素材数量；
- 导出状态、失败重试和授权提示；
- 记录每个素材组合的来源和版本。

**后端需求**

- 投放包生成任务；
- 标题、封面、钩子、评论文案结构化结果；
- 素材组合与测试分组；
- 导出文件打包与有效期；
- 商用授权记录；
- 投放数据回填和复盘建议；
- 素材内容安全与平台规格检查。

### 3.11 价格页

**页面目标**

解释会员、积分包、项目包和企业服务之间的差异。

**功能清单**

- 创作者会员；
- 专业会员；
- 团队会员；
- 积分充值包；
- 首集试制、连载和投放素材项目包；
- 企业咨询；
- 方案比较；
- 价格到个人中心权益的流程说明。

**前端开发要求**

- 套餐切换和对比；
- 根据登录状态显示当前方案；
- 购买按钮带入正确商品和来源；
- 价格、折扣和权益不得写死在页面脚本中。

**后端需求**

- 商品、价格、折扣、权益和上下架状态；
- 用户当前权益与升级差价；
- 活动资格和优惠计算；
- 项目包询价或下单规则。

### 3.12 确认订单与支付成功页

**页面目标**

完成商品确认、支付方式选择、支付和权益到账。

**功能清单**

- 商品摘要；
- 权益明细；
- 支付方式；
- 发票信息预留；
- 支付状态；
- 订单号；
- 权益到账结果；
- 返回个人中心。

**前端开发要求**

- 订单信息必须来自后端订单快照；
- 防止重复提交；
- 处理支付中、成功、失败、取消和超时；
- 支付结果通过后端查询确认，不只依赖前端跳转参数。

**后端需求**

- 创建订单、订单详情和取消订单；
- 支付渠道下单与支付通知；
- 支付状态查询和重复通知幂等；
- 权益发放、积分到账和失败补偿；
- 订单、支付、权益和发票记录关联；
- 对账与退款能力预留。

### 3.13 个人中心

**页面目标**

管理项目、项目档案、可复用资产、积分、订单、发票、授权和团队空间。

**功能清单**

- 用户资料与会员状态；
- 进行中项目；
- 项目档案；
- 角色、场景、道具、音色和封面资产；
- 积分钱包和消耗记录；
- 订单与发票；
- 商用授权；
- 团队空间；
- 继续上次创作。

**前端开发要求**

- 各模块分页和筛选；
- 资产预览、搜索和项目归属；
- 钱包流水可追溯到项目、任务和镜头；
- 订单、发票和授权支持查看与下载；
- 直接恢复最近项目的具体步骤。

**后端需求**

- 用户资料、会员和权益；
- 项目、项目档案和最近访问；
- 资产库、标签、版本和引用关系；
- 积分账户、冻结余额和流水；
- 订单、支付、发票和退款；
- 商用授权文件和下载记录；
- 团队、成员、角色和项目权限。

## 4. 运营后台功能与开发需求

### 4.0 运营后台模块总表

| 后台模块 | 管理对象 | 主要功能 | 对应前台页面 |
|---|---|---|---|
| 运营总览 | 全站经营数据 | 用户、项目、任务、支付、积分和导出指标 | 全站 |
| 客户旅程 | 用户与项目阶段 | 到达、转化、停留、阻塞和跟进 | 主流程全部页面 |
| 前后台连接矩阵 | 页面、动作和数据 | 功能覆盖、事件接收和接口健康状态 | 全站 |

- 今日访问、新增用户、新建项目；

- 支付金额、积分消耗、投放包导出；

### 4.2 客户旅程看板

- 用户当前阶段和下一步；
- 阻塞原因；
- 运营跟进记录。

**前端开发要求**

- 阶段漏斗和用户明细；
- 点击阶段查看用户、项目和事件；
- 阻塞标签和跟进状态；
- 支持按来源、套餐、题材和团队筛选。

**后端需求**

- 统一事件模型；
- 用户旅程阶段计算；
- 用户、项目、任务、订单事件关联；
- 阻塞规则和运营任务；
- 用户隐私和敏感信息脱敏。

### 4.3 前后台连接矩阵

**功能**

- 展示前台页面、用户动作、后台模块、智能能力和沉淀数据；
- 用于产品、研发和运营核对链路覆盖情况。

**前端开发要求**

- 支持筛选、搜索和导出；

**后端需求**

### 4.4 项目管理

**功能**

**前端开发要求**

- 多条件筛选和批量操作；
- 项目时间线；
- 阶段、版本、资产、任务和订单关联查看；

**后端需求**

- 项目搜索、分页、筛选和排序；
- 项目全量关系查询；
- 管理员修正状态和补偿操作；
- 操作审计和权限验证。

### 4.5 智能能力管理

**功能**

- 输入、输出、依赖和失败策略；

**后端需求**

**功能**

## 5. 前端公共开发需求

### 5.1 技术与结构

- 建议使用组件化前端框架正式重构当前静态原型；
- 建立统一页面布局、导航、按钮、卡片、表单、弹窗、表格和状态组件；
- 建立统一主题变量，支持日间和暗夜模式；
- 建立统一请求层、登录态刷新、错误处理和重试策略；
- 建立页面路由和权限路由；
- 正式环境不得在前端脚本中保存业务主数据、价格和权益规则。

### 5.2 状态设计

所有异步功能至少支持：

- 未开始；
- 加载中；
- 生成中；
- 待确认；
- 已完成；
- 已取消；
- 失败；
- 可重试；
- 积分不足；
- 待人工复核。

### 5.3 可用性

- 桌面、平板和手机响应式；
- 中文界面无供应商品牌名和无解释英文术语；
- 图片比例、裁切、懒加载、占位和替代文本统一；
- 表单支持键盘操作、焦点状态和错误定位；
- 颜色不作为唯一状态提示；
- 重要操作提供确认和可恢复路径。

### 5.4 事件埋点

统一记录：

- 页面访问；
- 来源渠道；
- 主要按钮点击；
- 登录完成；
- 项目创建；
- 阶段到达；
- 生成任务创建、成功和失败；
- 方向、大纲和资产确认；
- 投放包生成与导出；
- 价格查看、订单创建和支付；
- 个人中心回访和继续创作。

## 7. 非功能要求

### 7.1 性能

- 普通查询接口应有明确响应时间目标；
- 长时间智能生成必须异步执行；
- 图片和视频使用对象存储与内容分发；
- 大文件采用分片或直传方案；
- 运营指标使用聚合或缓存，避免实时扫描业务库。

### 7.2 稳定性

- 智能任务支持超时、重试、取消和死信处理；
- 支付和积分必须保证幂等；
- 关键任务状态可恢复；
- 外部供应商不可用时有降级、切换或人工复核方案；
- 关键服务有监控、告警和链路追踪。

### 7.3 安全与合规

- 登录、支付和后台管理接口需要严格鉴权；
- 用户手机号、企业信息和支付信息脱敏；
- 上传文件进行格式校验、病毒扫描和内容安全检查；
- 项目、资产和授权文件按权限访问；
- 临时下载链接有有效期；
- 管理员高风险操作写入不可抵赖审计日志；
- 数据保留、删除和用户注销规则需要明确。

### 7.4 可观测性

- 统一请求编号和任务编号；
- 记录接口耗时、错误率和调用量；
- 记录每项智能能力的成功率、耗时、成本和失败分类；
- 支持从用户、项目、任务、订单追踪完整链路；
- 运营后台能看到数据更新时间和异常状态。
