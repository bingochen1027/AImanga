# 创作平台用户分群、Customer Journey 与分层功能规划

## 1. 分群依据

平台不按“个人 / 企业”粗分创作体验，而按用户面对创作任务时的心态和控制欲分层。同一用户在不同项目中也可能切换层级。

| 层级 | 典型用户 | 进入时的心态 | 最担心什么 | 产品承诺 |
|---|---|---|---|---|
| Basic 傻瓜式 | 第一次做漫剧、内容副业者、只有想法的人 | “我不知道流程，先帮我做出来” | 术语太多、操作错误、花钱没结果 | 一次只问一个问题，AI 给推荐，先低成本看到第一版 |
| Medium 进阶式 | 有剧本经验、稳定更新的创作者、小团队主创 | “我知道我要什么，但不想从零搭流程” | AI 不可控、角色漂移、返工 | 控制关键创作变量，可比较、编辑、局部重做 |
| High 高手式 | 工作室、制片人、AI 导演、技术型创作者 | “我要控制工作流、成本、模型和批量交付” | 黑盒、无法复现、协作与成本失控 | Skill 可编排、版本可追溯、批量任务与模型路由可控 |

辅助业务分群：IP / 版权方关注改编一致性与授权；品牌方关注交付标准与投放；运营人员关注模板、Skill、任务、成本和合规。这些角色进入相同项目状态，但拥有不同权限和看板。

## 2. 使用心态 Customer Journey

| 阶段 | 用户场景 | Basic 心态 / 行动 | Medium 心态 / 行动 | High 心态 / 行动 | UX 引导 |
|---|---|---|---|---|---|
| 发现 | 浏览案例和模板 | “我也能做吗？”看同类作品 | 判断题材和工具是否适用 | 判断产线、模型和交付能力 | 案例展示成品，同时说明输入、耗时、成本和制作路径 |
| 开始 | 新建项目 | 选择“只有一个想法” | 上传剧本或套用模板 | 导入项目档案 / 工作流 | 三个起点，不先展示复杂编辑器 |
| 对齐 | 定义目标 | 回答单题卡片 | 批量确认平台、受众、题材 | 导入参数或编辑 Schema | AI 预填，未确认信息显式标记，不擅自补齐 |
| 发散 | 选择创作方向 | 看 3 张方向卡，选喜欢的 | 混合卡片要素、启用 DNA | 自定义发散维度和 Skill 参数 | 先选方向，再谈细节；不满意必须定位原因 |
| 决策 | 锁定大纲 | 对比两版并接受推荐 | 段落级合并、审稿建议勾选 | 版本分支、规则约束、批量评估 | 生成结果必须同时显示亮点、问题和风险 |
| 生产 | 资产 / 分镜 / 视频 | 使用推荐预设，先草稿 | 控制角色、镜头和局部重做 | 模型路由、并发、批量和失败策略 | 高成本动作前设置成本闸门，失败自动返还 |
| 审核 | 预览和修正 | 按问题清单逐项修复 | 定位回单镜头或资产版本 | 团队审核流、差异比较 | 不要求整集重做；每个问题可追溯到来源 |
| 发布 | 投放和合规 | 一键导出平台包 | 选择标题、封面、Hook 组合 | 批量版本、渠道矩阵和 API | 用平台语言说明规格，不暴露不必要模型术语 |
| 复盘 | 数据回流 | 接收下一集建议 | 比较素材表现并迭代 | 建立模板、自动化与 ROI 看板 | 表现数据回写为下一轮创作约束 |

## 3. 场景到功能拆分

### Basic：傻瓜式

- 一句话 / 上传剧本 / 模板三个入口
- 单题卡片式信息收集
- 推荐答案与“为什么推荐”
- 三张创作方向卡
- 双大纲简化对比
- AI 推荐版本
- 自动生成项目档案
- 推荐视觉预设与角色方案
- 草稿优先的视频生成
- 问题清单式修正
- 一键平台投放包
- 自动保存、版本恢复、成本预估

Basic 隐藏：模型名称、Prompt、Skill 编排、并发、Token 和高级镜头参数。

### Medium：进阶式

包含 Basic 的全部能力，并增加：

- 批量编辑核心信息
- 对标 DNA 的逐条确认
- 发散方向混合与局部重生成
- 大纲形式、差异轴和段落合并
- 四视角审稿及建议勾选
- 角色 / 场景候选对比和局部重绘
- 分镜粒度、逐镜时长、景别、运镜和对白
- 选择性视频升级与局部重做
- 多版本标题、封面和 Hook
- 项目版本、资产依赖和团队评论

### High：高手式

包含 Medium 的全部能力，并增加：

- Skill Manifest 与 Markdown 版本查看
- Skill 组合、条件分支和自定义检查点
- 项目档案 / DNA / 资产上下文选择
- Prompt 与结构化输入输出检查
- 模型路由、质量 / 速度 / 成本策略
- 批量任务、并发、重试、失败补偿策略
- 镜头级参数和负向 Prompt
- 工作流模板保存与团队共享
- 权限、审批、审计日志和 API / Webhook
- 项目预算、单集成本和模型成本分析

## 4. Streamlined UX 原则

1. **一条状态主线**：三种模式共用七阶段状态，不复制项目。
2. **渐进披露**：Basic 只见任务；Medium 增加控制面板；High 增加 Skill Inspector。
3. **一屏一主操作**：主按钮始终位于右上或任务底部，文案描述结果，例如“生成 3 个方向”。
4. **先解释结果，再显示技术**：用户先看会得到什么，再看 Skill 和模型。
5. **确认门槛统一**：信息确认、方案选择、资产锁定、成本确认、最终导出。
6. **回退不清空**：回到旧步骤创建新版本，保留已确认内容。
7. **建议可拒绝**：AI 推荐必须说明原因，用户可切换或自定义。
8. **问题优先**：每次生成都显示至少一个风险或待确认点。
9. **高成本延迟**：剧本与分镜确认前不批量生成视频。
10. **复杂度可升降**：模式切换只改变可见控制项，不改变底层结果。

## 5. 创作台信息架构

```text
全局工具栏
└── 项目工作台
    ├── 顶部：项目 / 保存 / 模式 / 成本 / 下一步
    ├── 左侧：七阶段项目产线
    ├── 中间：当前唯一任务
    │   ├── Basic 任务层
    │   ├── Medium 控制层
    │   └── High Skill / 模型层
    └── 右侧：AI Guide / 风险 / 输出说明
```

## 6. 模式切换规则

- 新用户默认 Basic。
- 上传完整剧本时建议 Medium，但不自动切换。
- 团队项目或打开 Skill Inspector 时建议 High。
- 从 High 切回 Basic 不删除高级设置，只折叠并显示“正在使用专业配置”。
- 每个项目记住上次使用模式。

## 7. 第一版 HTML 原型验证目标

- 可切换 Basic / Medium / High。
- 七阶段产线始终可见。
- 剧本策划内部可推进：核心信息 → 方向卡 → 大纲对比 → 审稿锁定。
- Medium 显示 DNA、大纲形式和版本控制。
- High 显示 Skill 版本、上下文、模型路由和预算策略。
- 每一步均有目标、风险、预计产物和明确主按钮。
- 状态在刷新后保留。

## 8. 后续页面改造

- 首页：案例卡补充适用人群、输入和制作成本。
- 产品功能：按七阶段和三种控制深度介绍，不按孤立功能介绍。
- 模板中心：标记 Basic / Medium / High 适用层级及预填内容。
- 案例中心：支持设为 DNA 参考或创建同类项目。
- 价格页：按 Skill 权限、视频额度、团队协作和专业控制拆套餐。
- 个人中心：项目档案、DNA、版本、Skill Run 和成本记录。
- Studio 总览：按“继续当前任务”排序，而不是只做 Kanban。
- Launch Kit：同时提供一键推荐、组合选择和批量矩阵三种控制层。
- Portal：管理 Skill Manifest、版本、权限、任务、失败补偿和审计。
