# 官网演示内容去重替换计划

目标：所有疑似来自 NeoTV / Neowow 的图片、标题、作者名、案例描述与项目名，统一替换为“虚构演示内容”。保留原有视觉气质与模块结构，但不复用现有图片、标题、人物、项目设定，降低重复与保密风险。

## 一、替换原则

1. 不再使用当前 `neo-cases` 目录中的案例图作为官网展示图。
2. 不再使用当前案例标题、作者名、项目名、剧情关键词。
3. 风格保留：竖屏漫剧感、蓝紫渐变、电影感构图、强情绪封面、短剧投放包装。
4. 内容改为虚构演示案例，不指向真实项目、真实作者或真实客户。
5. 图片尽量不包含文字，标题与描述由 HTML 独立承载，方便后续改文案。

## 二、需要替换的页面与内容

| 页面 | 当前风险内容 | 替换方式 |
|---|---|---|
| 首页 `frontend/index.html` | Banner 主视觉、案例卡片标题、案例图、作者名、标签 | 换成 8 个虚构演示案例，保留案例阵列布局 |
| 案例中心 `frontend/showcase.html` | 8 个 NeoTV 风格案例图、标题、描述、作者名、数据 | 换成完整虚构案例库，结构不变 |
| 个人中心 `frontend/account.html` | 项目名、最近创作图、资产图、授权记录 | 换成用户虚构项目与资产 |
| 登录页 `frontend/login.html` | 拼贴案例图 | 换成 3 张虚构演示封面 |

## 五、HTML 替换顺序

1. 先替换首页与案例中心：这是最明显的公开展示入口。
2. 再替换个人中心与登录页：避免内部项目名继续露出。
3. 再替换投放包与创作指引：把演示链路补完整。
4. 最后清理旧图片引用：确保页面不再引用 `assets/images/neo-cases/`。

## 六、后续执行节奏

1. 确认本计划的新案例标题与风格方向。
2. 生成第一批 8 张主案例封面。
3. 预览图片，挑选需要重生的图。
4. 将通过的图片放入 `frontend/assets/images/story-cases/`。
5. 替换 HTML 中的图片、标题、描述、作者名与示例数据。
6. 本地打开页面检查：首页、案例中心、个人中心、登录页、抖音投放包、创作指引。
