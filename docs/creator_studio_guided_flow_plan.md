# 创作工作台引导流程规划 v1.0

## 1. 产品目标

创作工作台不是把多个 AI 按钮堆在一起，而是把 Neowow 的 Markdown Skill 转译成普通创作者能理解、能确认、能回退的分阶段流程。

核心体验目标：

- 新手始终知道“我现在在哪一步、为什么做这一步、需要提供什么”。
- AI 每次只要求一个清晰决策，不一次抛出整份复杂表单。
- 每次生成前说明输出物、预计耗时和积分；确认后才执行高成本任务。
- 所有 Skill 输出可编辑、可对比、可锁定、可回退，并保留版本。
- 专业用户可以切换“快速模式”，但底层仍记录相同的阶段状态。

## 2. 两层引导结构

### 2.1 外层：项目制作产线

外层左侧阶段栏固定显示整个项目进度：

1. 剧本策划
2. 视觉设定
3. 角色 / 场景 / 道具
4. 分镜脚本
5. 视频生成
6. 后期与预览
7. 发布与投放

每个阶段只有四种状态：`未开始`、`进行中`、`待确认`、`已锁定`。生成失败、积分不足和人工审核作为状态附加标记，不增加新阶段。

### 2.2 内层：当前 Skill 微向导

每个外层阶段加载一个或多个 Neowow Markdown Skill。Markdown 不直接展示给用户，而是被转换为：

- 当前目标
- 必填输入
- AI 执行动作
- 输出预览
- 风险与问题
- 用户确认点
- 下一步条件

剧本策划阶段使用上传的《剧本创作 Skill v2.0》，内部保留原始七步逻辑，但通过卡片、选择器、对比视图和确认按钮呈现。

## 3. 工作台整体布局

### A. 一级工具栏（最左侧，窄栏）

- 首页
- 制作工作台
- Skill 工具箱
- 我的项目
- 资产管理
- 账户与积分

### B. 项目阶段栏（左侧，约 240px）

- 项目名称与返回按钮
- 七个制作阶段
- 每阶段状态、异常和完成标记
- 项目总体进度
- “保存并退出”与分享协作

### C. 主任务区（中间，自适应）

只展示当前阶段的一个主要任务。页面结构固定为：

1. 当前目标与完成标准
2. 输入或选择区域
3. AI 生成 / 分析结果
4. 对比与编辑
5. 风险、问题和创作笔记

### D. AI Guide（右侧，可收起，约 320px）

- 用新手语言解释当前步骤
- 显示“为什么要做”
- 推荐选项及原因
- 当前缺失信息
- Skill 名称、版本和运行状态（高级信息折叠）
- 可随时提问，但回答必须保持在当前阶段上下文中

### E. 顶部操作栏

- 自动保存状态
- 当前输出规格
- 本步预计积分 / 时间
- 返回上一步
- 主按钮：“确认并进入下一步”

## 4. 新建项目与模式选择

### Step 0：选择起点

三个入口：

1. **只有一个想法**：进入剧本策划 Step 1。
2. **已有剧本**：粘贴或上传，AI 先解析，再让用户补齐缺失信息。
3. **从模板 / 案例开始**：预填题材、平台、视觉方向；案例可作为“对标 DNA”候选，但必须由用户确认。

首次用户只显示“引导模式”；完成过一个项目后允许选择：

- 引导模式：逐步解释与确认。
- 快速模式：合并低风险步骤，仍保留成本闸门和最终锁定。

## 5. 阶段一：剧本策划 Skill 的七步交互

### 1. 核心信息收集

**界面**：一题一卡，不展示长表单；顶部显示“1 / 5”。

依次询问：作品时长、平台、目标受众、题材、核心情绪。每题提供常用快捷选项和“自定义”。画风未明确时，在进入下一步前补问；有对标作品时触发 DNA 流程。

**已有剧本分支**：上传后自动提取答案，用户逐项确认；不确定的字段显示“需要你确认”，不得自动编造。

**完成条件**：五个核心字段完整，身份类型已确定。

### 2. 发散方向卡

**界面**：一次生成 3–5 张方向卡，可横向对比。

每张卡固定显示：一句话定位、主角、冲突、卖点、风险、难度。用户可以：

- 选择一个方向
- 收藏某张卡的局部要素
- 要求重新发散
- 输入具体不满意维度

最多无条件重做两次；第三次必须回到问题诊断，不继续盲目生成。

### 3. 大纲方案

#### 3a. 自动推荐形式

系统根据时长推荐：故事梗概、千字大纲或分集大纲，并明确解释推荐原因。用户可更换形式。

随后选择差异轴：

- 稳健版 vs 大胆版
- 主角驱动版 vs 事件驱动版

#### 3b. 双版本对比

**界面**：左右双栏或移动端上下对比。共同字段对齐展示，差异处高亮。

每版必须包含人物配置、结构、三条“为什么这样写”、风险预警和创作笔记。用户可以整版选择，也可以组合两版中的段落。若启用对标 DNA，明确显示引用了哪些已确认规律。

**完成条件**：选定一个版本或完成组合版。

### 4. 是否进入多视角审稿

以成本和收益明确的选择卡呈现：

- 推荐：四视角审稿
- 跳过：直接局部修改 / 锁定

跳过时提示可能遗漏结构、制作、观众和平台风险，但不阻止继续。

### 5. 四视角顾问审稿

四个页签：编剧、制片、观众、平台。每个视角至少展示两个真实问题及建议，不允许只有表扬。

系统汇总为按优先级排序的修改清单。用户逐条勾选“采纳 / 暂不采纳”，右侧实时预览修改影响，确认后生成更新版。最多进行两轮审稿，第二轮自动切换关注维度。

### 6. 最终锁定

锁定前检查：

- 未解决的必改问题
- 未确认的自动推导信息
- 与对标 DNA 冲突的内容
- 下游资产生成所需字段是否齐全

“锁定大纲”是明确操作。锁定后生成版本号；再次修改会创建新版本，不覆盖旧版本。

### 7. 生成项目档案

将 Skill 规定的项目档案转换为可视化档案页，同时支持：

- 在线分区浏览
- 下载 Word 友好文本
- 复制全文
- 查看待补充清单
- 作为下游角色、场景、分镜 Skill 的统一上下文

完成后，外层“剧本策划”变为已锁定，并进入视觉设定。

## 6. 对标 DNA 作为可插拔支线

DNA 不是外层主阶段，而是可在任何剧本步骤触发的支线抽屉。

交互顺序：

1. 用户输入或从案例选择对标作品。
2. AI 给出 5–7 条规律、观察依据和置信度。
3. 用户逐条保留、修改、删除或补充。
4. 确认后的规律成为项目约束。
5. 返回触发前的剧本步骤。

后续每次生成都在“创作笔记”中显示至少一条实际引用的 DNA 规律。AI 不熟悉作品时必须明确说明，改为让用户提供关键特征。

## 7. 后续外层阶段规划

### 阶段二：视觉设定

输入：画风、比例、平台、时长、清晰度、节奏、参考图。

交互：预设卡 + 自定义；生成视觉圣经草案；并排对比 2–3 种方向；锁定后才进入资产生成。

输出：视觉风格规范、色彩、镜头偏好、负面约束、视频规格。

### 阶段三：角色 / 场景 / 道具

输入直接继承项目档案。先显示 AI 从剧本识别出的资产清单，再让用户决定“必须生成 / 可复用 / 暂不需要”。

交互：逐资产生成、候选图对比、局部重绘、一致性评分、确认锁定。角色和场景缺少必要视图时不能直接进入视频阶段。

输出：角色圣经、场景四视图、关键道具、音色和复用规则。

### 阶段四：分镜脚本

先选择分镜粒度：精简、标准、细拆。AI 生成镜头表后，用户可逐镜编辑时长、景别、运镜、对白、画面提示词和负向提示词。

交互：镜头卡拖动排序、批量调整、风险标记、前 3 秒和集尾钩子检查。分镜锁定后才允许批量生成视频。

输出：镜头表、故事板、视频 Prompt、预计镜头数量与成本。

### 阶段五：视频生成

生成前显示模型路由、画质、预计时间和积分。支持先生成低成本草稿，再选择镜头升级为标准或投放级。

交互：任务队列、逐镜状态、失败原因、自动重试、失败积分返还、替换模型。用户无需理解底层模型参数，只在“质量 / 速度 / 成本”三种偏好中选择。

输出：逐镜视频、版本、生成记录和成本账单。

### 阶段六：后期与预览

自动合成镜头、配音、字幕、音乐和转场。提供整集预览及问题列表。

交互：时间线简化编辑、对白与字幕修正、声音替换、一致性检查、合规初检。问题可定位回角色、分镜或单镜头生成，不要求整集重做。

输出：第一集母版、字幕、音轨、修改记录。

### 阶段七：发布与投放

调用投放素材与合规 Skill，生成标题、封面、前 3 秒 Hook、预告、多版本测试组合、AI 生成说明和商用授权记录。

交互：选择目标平台、预览平台安全区、勾选导出素材、生成备案材料。结果回流到投放复盘。

输出：平台成片、投放包、合规包和导出记录。

## 8. 所有 Skill 的统一接入规范

每份 Neowow Markdown Skill 进入产品前，需注册为统一 Skill Manifest：

- Skill 名称与唯一 Key
- Markdown 版本与更新时间
- 适用阶段
- 必填 / 可选输入
- 分支条件
- 输出 Schema
- 用户确认点
- 可回退范围
- 预计积分与耗时
- 失败和信息不足处理
- 是否生成创作笔记
- 是否引用项目档案 / DNA / 已锁定资产

Portal 需要保存 Skill 版本、启停状态、套餐权限、积分规则、成功率、平均耗时和每次运行所使用的版本。项目必须记录“哪一个输出由哪一版 Skill 生成”。

## 9. 全站页面内容调整顺序

### P0：先打通创作主链

1. `wizard.html`：改为 Step 0 新建项目和入口选择。
2. `studio.html`：改为项目总览 + “继续当前步骤”，不再承担编辑器本体。
3. `editor-guide.html`：升级为真正的分阶段创作工作台 Demo。
4. `launch-kit.html`：成为外层阶段七的投放工作区。

### P1：同步用户理解

5. `features.html`：用七阶段产线解释 Skill，不再堆功能卡。
6. `templates.html`：模板直接预填 Step 0 / Step 1，而不是普通内容卡跳转。
7. `showcase.html`：案例增加“设为对标 DNA / 从此案例开始”的明确关系，Demo 阶段可只展示状态。
8. `account.html`：增加项目档案、DNA 报告、版本历史、Skill 运行记录。

### P2：商业与运营闭环

9. `pricing.html`：按阶段解释套餐权益、Skill 次数和视频成本。
10. `checkout.html` / `payment-success.html`：说明购买后解锁哪些阶段和生成额度。
11. `portal/index.html`：增加 Skill Manifest、版本、任务、失败补偿、项目阶段和人工审核。

## 10. 关键状态与事件

建议最小状态模型：

```text
Project
  current_stage
  current_micro_step
  mode: guided | fast
  status: draft | active | review | locked | blocked
  project_dossier_version
  dna_rules_version
  credits_estimate

SkillRun
  skill_key
  skill_version
  input_snapshot
  output_version
  status
  credits_cost
  error_reason
  user_decision
```

关键事件：`stage_started`、`input_confirmed`、`skill_run_started`、`output_generated`、`revision_requested`、`output_locked`、`stage_completed`、`stage_reopened`、`credits_refunded`。

## 11. Demo 第一轮实现范围

第一轮只做一条可点击主链，不模拟所有复杂能力：

1. Step 0 选择“一句话创作 / 上传剧本 / 模板开始”。
2. 剧本 Skill Step 1 五项信息收集。
3. Step 2 三张方向卡选择。
4. Step 3 双大纲对比与锁定。
5. Step 5 审稿建议勾选。
6. Step 7 项目档案预览。
7. 外层阶段栏状态更新并进入视觉设定占位页。

这条链路完成后，再接角色资产、分镜和视频 Skill。否则一次铺开所有 Skill，会得到很多漂亮页面，但无法验证新手是否真的走得通。

## 12. 验收标准

- 新用户不看说明文档，也能说出当前步骤和下一步。
- 每屏只有一个主操作。
- 所有高成本生成前都有成本确认。
- 所有 AI 生成结果至少有“问题 / 风险”区，不只展示优点。
- 用户可回到任意已完成步骤，旧版本不丢失。
- 刷新或退出后可以继续当前微步骤。
- Skill Markdown 更新后，旧项目仍能追溯原版本。
- 从创建项目到锁定剧本档案，Demo 可在 5–8 分钟内完成。
