# 智能漫剧后台第一阶段：只读数据看板

## 目标

第一阶段以数据查看为主，仅开放用户与权限管理，回答四个问题：

1. 今天是否有人注册并开始创作？
2. 项目目前进行到哪一步？
3. 哪些生成任务失败或进入待复核状态？
4. 用户是否有足够权益继续完成第一次生成？

## 功能优先级

| 优先级 | 模块 | 保留原因 | 第一阶段范围 |
|---|---|---|---|
| 必做 | 数据概览 | 快速判断主流程是否健康 | 访问、新用户、新项目、异常记录、最新项目 |
| 必做 | 项目记录 | 了解用户当前创作阶段 | 项目、用户、阶段、进度和状态，只读 |
| 必做 | 生成记录 | 观察智能生成是否稳定 | 生成中、失败和待复核记录，只读 |
| 必做 | 用户与积分记录 | 了解试用和权益状态 | 用户、联系方式、方案、可用积分，只读 |
| 必做 | 最近数据记录 | 确认前台数据是否成功进入后台 | 页面访问、注册、项目创建和生成事件 |
| 必做 | 今日与昨日对比 | 判断核心指标趋势 | 访问、新用户、新项目和生成任务的数量与变化率 |
| 必做 | 用户与权限管理 | 控制谁可以使用平台或进入后台 | 角色调整、用户启用与禁用、全部用户字段导出 |
| 下一阶段 | 异常处理与工单 | 数据稳定后再建立处理闭环 | 工单、负责人、处理状态和备注 |
| 下一阶段 | 智能能力配置 | 生产稳定后才需要复杂配置 | 版本、计费、路由、灰度和回滚 |
| 下一阶段 | 订单与支付 | 商业化阶段接入 | 商品、订单、支付、退款、发票 |
| 下一阶段 | 合规与授权 | 正式发布和商用交付需要 | 内容风险、审核、授权文件 |
| 以后再做 | 完整客户旅程 | 数据规模形成后再做漏斗分析 | 阶段转化、停留、阻塞和跟进 |
| 以后再做 | 投放复盘 | 有真实投放数据后再建设 | 素材组合、效果归因、下一集建议 |
| 以后再做 | 前后台连接矩阵 | 研发管理工具，不影响用户闭环 | 功能覆盖和接口健康检查 |
| 以后再做 | 复杂财务分析 | 交易规模提升后再建设 | 对账、利润、成本分摊和财务报表 |

## 最小数据对象

- 用户：编号、手机号、当前方案、可用积分；
- 项目：编号、名称、用户、当前阶段、进度、状态；
- 任务：编号、项目、任务类型、状态、积分、失败原因；
- 用户事件：注册、创建项目、创建生成任务；
- 权益流水：试用发放、任务预扣、成功扣减和失败返还。
- 用户权限：用户角色、账号状态和后台访问范围。

## 最小后端接口

1. 运营总览指标查询；
2. 最新项目状态查询；
3. 项目记录列表查询；
4. 生成任务记录列表查询；
5. 用户基本信息与积分查询；
6. 前台访问、注册、项目创建和任务事件接收；
7. 最近事件记录查询。
8. 用户列表与完整字段查询；
9. 用户启用和禁用；
10. 用户角色更新；
11. 用户数据导出。

## 暂不进入第一阶段的能力

- 复杂图表和多维分析；
- 完整会员运营；
- 模板与内容管理；
- 智能能力在线编排；
- 多供应商配置；
- 合规规则配置；
- 投放渠道接口；
- 发票、合同和复杂财务；
- 批量运营触达。
- 异常处理、任务重试和人工复核；
- 工单创建、分配、处理和关闭；
- 人工积分补偿和业务状态修改；

## 第一阶段验收

- 前台注册后，后台新增用户数量变化；
- 前台创建项目后，后台能看到项目和当前阶段；
- 前台创建生成任务后，后台能看到任务状态；
- 失败任务可以查看状态，但没有重试入口；
- 用户积分不足时后台能够识别；
- 页面不存在工单、补偿、复核和任务重试入口；
- 用户可以按角色授权，并支持启用或禁用；
- 导出的表格包含用户编号、名称、手机号、邮箱、角色、状态、方案、积分、项目数、注册时间、最近活跃和来源；
- 数据概览中的核心指标同时显示昨天数值和变化率；
- 页面为单一数据看板，移动端无横向页面溢出。
